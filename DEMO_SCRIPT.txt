[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture.
First, every query enters through the security_checkpoint node which filters out PII and malicious injection attempts. (pause)
If clean, the request is passed to the orchestrator agent. (pause)
The orchestrator delegates work to the invoice_auditor sub-agent, which utilizes our local MCP server to query purchase orders and inventory logs. (pause)
Next, any mismatches are analyzed by the discrepancy_resolver sub-agent to determine if the variation is within tolerance. (pause)
Our router_node then decides: if the discrepancy is under $50, it goes to the finalize_invoice node for automatic approval; if it's $50 or more, it hits the get_user_approval node, pausing the workflow to wait for manual manager sign-off. (pause)
